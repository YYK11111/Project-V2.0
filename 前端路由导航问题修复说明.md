# 前端路由导航问题修复说明

## 修复日期
2026-04-10

## 问题描述

### 问题1: 首页路径重复
访问系统时，浏览器地址栏显示 `http://localhost:1994/index/index`，路径中出现重复的 "index"。

**根本原因：**
- "首页"一级菜单的 path 为 `index`
- "首页"有一个隐藏的子菜单，path 也是 `index`
- 前端路由转换时，子菜单 path 被拼接为 `/index/index`
- catalog 菜单的 redirect 指向了这个重复的路径

### 问题2: 点击"项目管理"报错
点击"项目管理"菜单时，浏览器控制台抛出 `Uncaught RangeError: Maximum call stack size exceeded` 错误。

**根本原因：**
- 一级 catalog 菜单在 `SidebarItem.vue` 中被强制当作叶子菜单处理
- 但 `isLeafMeu` 方法返回 true 时，没有设置 `onlyOneChild` 变量
- 导致模板中访问 `onlyOneChild.path` 时为 null，引发错误
- Vue Router 在解析错误的路径时可能产生循环重定向

## 修复方案

### 1. 前端代码修复

#### 1.1 修复 SidebarItem.vue
**文件：** `/nest-admin-frontend/src/layout/components/Sidebar/SidebarItem.vue`

**修改内容：**
- 在 `isLeafMeu` 方法中，为一级 catalog 菜单设置 `onlyOneChild`
- 优先使用菜单的 `redirect` 字段
- 如果没有 redirect，则使用第一个可见子路由
- 如果所有子路由都隐藏或没有子路由，使用菜单自身的 path

```javascript
if (item.type === 'catalog' && this.level == 1) {
  // 为一级 catalog 设置 onlyOneChild，使用其 redirect 或第一个子路由
  if (!this.onlyOneChild) {
    if (item.redirect) {
      this.onlyOneChild = { ...item, path: item.redirect }
    } else if (item.children && item.children.length > 0) {
      const firstVisible = item.children.find(c => !c.isHidden)
      if (firstVisible) {
        this.onlyOneChild = firstVisible
      } else {
        this.onlyOneChild = { ...item, path: item.path }
      }
    } else {
      this.onlyOneChild = { ...item, path: item.path }
    }
  }
  return true
}
```

**同时添加空值检查：**
- 模板中使用 `v-if="isLeafMeu(route) && onlyOneChild"`
- 在 `isLeafMeu` 中访问 `onlyOneChild.noShowingChildren` 前先检查 null

#### 1.2 优化 routes.js 路由转换逻辑
**文件：** `/nest-admin-frontend/src/router/routes.js`

**修改内容：**

1. **简化 redirect 设置逻辑**（第234-256行）
   - 直接使用子路由的绝对路径作为 redirect
   - 移除冗余的路径判断和拼接逻辑

```javascript
if (visibleChild) {
  // 直接使用子路由的绝对路径作为 redirect
  route.redirect = visibleChild.path
}
```

2. **优化一级目录的所有子路由都隐藏时的处理**
   - 一级目录：redirect 到自身
   - 二级目录：redirect 到第一个子路由

```javascript
if (level === 1) {
  route.redirect = route.path
} else {
  const firstChild = route.children[0]
  if (firstChild) {
    route.redirect = firstChild.path
  }
}
```

3. **增强根路径重定向的安全性**（第156-163行）
   - 在设置 redirect 前验证路由数据存在

```javascript
if (userRoutes && userRoutes.length > 0) {
  const firstRoute = userRoutes[0]
  const rootRoute = router.getRoutes().find((e) => e.path == '/')
  if (rootRoute && firstRoute) {
    rootRoute.redirect = firstRoute.path
  }
}
```

### 2. 数据库修复

#### 2.1 清除一级 catalog 菜单的 component 字段
**执行SQL：**
```sql
UPDATE sys_menu SET component = NULL 
WHERE id IN ('26', '27', '28', '29') AND type = 'catalog';
```

**影响菜单：**
- 项目管理 (id=26)
- 任务管理 (id=27)
- 工单管理 (id=28)
- 文档管理 (id=29)

**原因：** 一级 catalog 菜单不应有 component 字段，由前端自动设置为 Layout 组件。

#### 2.2 删除"首页"的重复子菜单
**执行SQL：**
```sql
DELETE FROM sys_menu WHERE id = '19' AND parent_id = '18';
```

**原因：** 
- "首页"一级菜单 path 为 `index`
- 其子菜单 path 也是 `index`，导致路径拼接后变成 `/index/index`
- 删除隐藏的子菜单后，一级 catalog 菜单会自动 redirect 到自身 `/index`

## 技术要点

### 1. 一级 Catalog 菜单的处理规则
- type = 'catalog' 且 level = 1
- component 字段应为 NULL（后端不设置）
- 前端自动设置为 Layout 组件
- 如果有子菜单，redirect 到第一个可见子菜单
- 如果没有子菜单或所有子菜单都隐藏，redirect 到自身

### 2. 路径拼接规则
- 顶级菜单：path 必须以 `/` 开头
- 子菜单：相对路径会与父路径拼接
- 已包含父路径前缀的路径不再重复拼接
- 最终所有路由的 path 都是绝对路径

### 3. 空值防护
- 访问 `onlyOneChild` 属性前必须检查是否为 null
- 使用可选链操作符 `?.` 或条件判断
- 确保所有代码路径都有合适的 fallback
