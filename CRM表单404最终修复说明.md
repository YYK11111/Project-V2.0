# CRM表单404问题最终修复

## 问题描述
访问 `http://localhost:1994/crm/customer` 后，点击"新建客户"按钮跳转到 `http://localhost:1994/404`

## 根本原因

### 菜单结构问题
之前的CRM菜单结构是3层嵌套：
```
一级（catalog）: CRM客户管理 (/crm)
  └─ 二级（menu）: 客户管理 (customer, component: index.vue)
      └─ 三级（menu, hidden）: 客户表单 (form, component: form.vue)
```

**问题分析**：
- 二级菜单是 `menu` 类型，会直接加载 `index.vue` 组件
- `menu` 类型的路由不会渲染 `<RouterView>`
- 当访问 `/crm/customer/form` 时，由于父路由没有 `<RouterView>`，子路由无法渲染
- 导致404错误

### 正确的结构（参照项目管理）
项目管理模块的正确结构：
```
一级（catalog）: 项目管理 (projectManage, component: index.vue)
  ├─ 二级（menu）: 项目列表 (projectManage, 实际不显示)
  └─ 二级（menu, hidden）: 项目表单 (projectManage/form, component: form.vue)
```

**关键点**：
- 列表页和表单页都是**一级菜单的直接子路由**
- 表单页的 path 包含完整路径：`projectManage/form`
- 这样访问 `/projectManage/form` 时，一级 catalog 会使用 Layout，可以正确渲染子路由

## 解决方案

将CRM改为与项目管理相同的结构：

### 修复后的菜单结构
```
一级（catalog）: CRM客户管理 (/crm)
  ├─ 客户管理 (menu, customer, component: index.vue)
  ├─ 客户表单 (menu, hidden, customer/form, component: form.vue)
  ├─ 互动记录 (menu, interaction, component: index.vue)
  ├─ 互动表单 (menu, hidden, interaction/form, component: form.vue)
  ├─ 销售机会 (menu, opportunity, component: index.vue)
  ├─ 机会表单 (menu, hidden, opportunity/form, component: form.vue)
  ├─ 合同管理 (menu, contract, component: index.vue)
  └─ 合同表单 (menu, hidden, contract/form, component: form.vue)
```

## 技术要点

### Vue Router嵌套路由规则

**catalog类型**：
- 作为路由容器
- 一级catalog使用 Layout 组件
- 会自动渲染 `<RouterView>` 来显示子路由

**menu类型**：
- 直接加载指定的组件
- 如果有子路由，需要确保父级能够渲染子路由
- 如果父级也是 menu 类型且直接加载组件，则子路由无法显示

### 路径拼接规则

当前端调用 `goRoute(null, 'form')` 时：
```javascript
const fullPath = path.startsWith('/') ? path : this.$route.path + '/' + path
```

- 当前路径：`/crm/customer`
- 目标路径：`form`
- 拼接结果：`/crm/customer/form` ✅

### 为什么之前的结构不行

**错误的3层结构**：
```
/catalog (Layout)
  └─ /catalog/menu (index.vue)  ← 这里没有RouterView
      └─ /catalog/menu/form (form.vue)  ← 无法渲染
```

**正确的2层结构**：
```
/catalog (Layout)
  ├─ /catalog/menu (index.vue)
  └─ /catalog/menu/form (form.vue)  ← Layout有RouterView，可以渲染
```

## 注意事项

1. **所有带表单页的模块都应采用此结构**
2. **表单页的path必须包含完整路径**（如 `customer/form` 而不是 `form`）
3. **表单页必须设置为隐藏**（is_hidden = 1）
4. **按钮权限可以挂在列表页下**（作为其子菜单）

## 对比总结

| 特性 | 错误结构 | 正确结构 |
|------|---------|---------|
| 层级 | 3层（catalog → menu → menu） | 2层（catalog → menu） |
| 表单path | `form` | `customer/form` |
| 表单parent_id | 指向列表页ID | 指向catalog ID |
| RouterView | ❌ 列表页没有 | ✅ Layout有 |
| 结果 | 404错误 | 正常显示 |
