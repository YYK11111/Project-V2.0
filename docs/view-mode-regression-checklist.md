# 查看视图回归验收清单

## 目的

本清单用于回归验证业务页面的“查看视图”是否已经从“禁用表单控件”切换为“只读展示组件”，并确认没有影响原有编辑、保存、审批等功能。

## 通用检查项

每个页面都按以下项目检查：

1. 通过列表页“查看”按钮进入页面，地址栏应带 `action=view`
2. 查看页标题应显示“查看xxx”而不是“编辑xxx”
3. 查看态不应出现禁用输入框、禁用下拉、禁用人员选择器作为主要展示方式
4. 关联对象不应显示 id，应显示名称、编号、头像等业务信息
5. 空值统一显示 `-`
6. 状态、优先级、审批状态、阶段等应显示为标签或明确文本
7. 富文本应为只读内容块，不应显示禁用编辑器
8. 附件应显示为文件列表或文件链接，不应显示上传控件
9. 点击返回后应能正常回到列表页
10. 从查看页切到编辑页后，保存逻辑应保持正常

## 第一批页面

### 1. 项目管理 `projectManage`

检查项：

1. 项目名称、项目编号是否正常显示
2. 客户是否显示客户名称，而不是 `customerId`
3. 项目负责人是否显示头像和姓名
4. 优先级、项目状态是否显示为标签
5. 开始时间、结束时间、预算、实际成本、进度是否正常显示
6. 项目成员表格中：
   - 成员是否显示头像和姓名
   - 角色是否显示文字
   - 核心成员是否显示“是/否”
7. 里程碑计划表格中：
   - 里程碑名称、计划完成日期、状态、交付物、描述、排序是否正常显示
8. 项目描述是否为只读富文本块
9. 项目附件是否显示文件列表

### 2. 任务管理 `taskManage`

检查项：

1. 任务编号是否正常显示，不应显示“系统自动生成”
2. 所属项目是否显示项目名称
3. 父任务是否显示名称
4. 负责人是否显示头像和姓名
5. 经办人是否显示多人头像和姓名列表
6. 状态、优先级、审批状态是否显示为标签
7. 任务描述、验收标准是否显示为只读富文本
8. 任务附件是否显示为文件列表
9. 历史任务若原来没有编号，打开查看页后是否已经自动补号

### 3. 工单管理 `ticketManage`

检查项：

1. 所属项目是否显示项目名称
2. 关联任务是否显示任务名称
3. 提交人、处理人是否显示头像和姓名
4. 状态、优先级、审批状态是否显示为标签
5. 工单内容、重现步骤、根因分析、解决方案是否显示为只读富文本
6. 工单附件是否显示为文件列表

### 4. 变更管理 `changeManage`

检查项：

1. 所属项目是否显示项目名称
2. 申请人、审批人是否显示头像和姓名
3. 变更状态、审批状态是否显示为标签
4. 变更原因、变更描述、影响分析是否正常显示
5. 成本影响、进度影响、排序是否正常显示

### 5. 文档管理 `documentManage`

检查项：

1. 文档名称、版本号、文档类型是否正常显示
2. 所属项目是否显示项目名称
3. 上传人是否显示头像和姓名
4. 文档内容摘要是否正常显示
5. 文件链接是否可点击

## 第二批页面

### 6. 风险管理 `riskManage`

检查项：

1. 所属项目是否显示项目名称
2. 责任人是否显示头像和姓名
3. 风险分类、风险等级、风险状态是否正常显示
4. 风险状态是否显示为标签
5. 风险描述、应对措施、影响程度、日期、排序是否正常显示

### 7. 用户故事 `userStoryManage`

检查项：

1. 所属项目是否显示项目名称
2. 父级故事是否显示标题
3. 所属 Sprint 是否显示 Sprint 名称
4. 负责人是否显示头像和姓名
5. 类型、状态是否正常显示
6. 状态是否显示为标签
7. 故事点、优先级、预估日期、描述、验收标准是否正常显示

### 8. Sprint 管理 `sprintManage`

检查项：

1. Sprint 名称、所属项目是否正常显示
2. Sprint 状态是否显示为标签
3. Sprint 目标、开始日期、结束日期是否正常显示
4. 总故事点、已完成故事点、总任务数、已完成任务数是否正常显示

### 9. 里程碑管理 `milestoneManage`

检查项：

1. 里程碑名称、所属项目、计划完成日期是否正常显示
2. 状态是否显示为标签
3. 里程碑描述是否正常显示
4. 交付物清单是否以 tag 列表形式显示
5. 排序是否正常显示

## 第三批页面（CRM）

### 10. 客户管理 `crm/customerManage`

1. 客户名称、简称、信用代码、行业、规模、地址是否正常显示

3. 客户状态、审批状态是否显示为标签
4. 销售负责人是否显示头像和姓名
5. 所属部门是否显示部门名称，而不是 `deptId`
6. 客户价值、客户描述是否正常显示

### 11. 合同管理 `crm/contractManage`

检查项：

1. 合同名称、合同编号是否正常显示
2. 客户是否显示客户名称和编号
3. 合同负责人是否显示头像和姓名
4. 合同金额、已收款金额、签订时间、开始时间、结束时间是否正常显示
5. 合同状态是否显示为标签
6. 合同文件是否显示为可点击链接
7. 备注是否正常显示

### 12. 销售机会 `crm/opportunityManage`

检查项：

1. 机会名称、机会编号是否正常显示
2. 客户是否显示客户名称和编号
3. 销售负责人是否显示头像和姓名
4. 销售阶段是否显示为标签
5. 预期金额、成功概率、预计成交时间是否正常显示
6. 机会描述、失败原因是否正常显示

### 13. 客户互动 `crm/interactionManage`

检查项：

1. 客户是否显示客户名称和编号
2. 互动类型是否正常显示
3. 互动类型是否显示为标签或明确文本
4. 互动时间、互动内容、互动人、下次跟进时间是否正常显示

## 编辑态回归检查

每批页面除了查看态，还应抽样检查编辑态：

1. 从列表页点击“编辑”进入，不应变成查看态
2. 表单控件应仍可编辑
3. 提交后能正常保存
4. 已有审批流页面：
   - 提交审批按钮仍可用
   - 审批状态流转展示正常

## 建议回归顺序

建议按下面顺序逐页验证：

1. `projectManage`
2. `taskManage`
3. `ticketManage`
4. `changeManage`
5. `documentManage`
6. `riskManage`
7. `userStoryManage`
8. `sprintManage`
9. `milestoneManage`
10. `crm/customerManage`
11. `crm/contractManage`
12. `crm/opportunityManage`
13. `crm/interactionManage`

## 记录建议

建议回归时按以下格式记录：

1. 页面名称
2. 查看入口是否正常
3. 是否仍出现禁用控件
4. 关联对象是否显示名称/头像
5. 富文本/附件是否正常
6. 编辑态是否回归通过
7. 备注问题
