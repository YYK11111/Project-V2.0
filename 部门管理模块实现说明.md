## 概述

## 使用方法

### 在其他页面中使用部门选择

```vue
<script setup>
import { getTrees as getDeptTrees } from '@/views/system/depts/api'

const deptList = ref([])

// 获取部门树并展平
getDeptTrees().then((res) => {
  const flattenDepts = (depts, result = []) => {
    depts.forEach(dept => {
      result.push(dept)
      if (dept.children && dept.children.length > 0) {
        flattenDepts(dept.children, result)
      }
    })
    return result
  }
  deptList.value = res.data ? flattenDepts(res.data) : []
})
</script>

<template>
  <el-select v-model="form.deptId" placeholder="请选择部门">
    <el-option 
      v-for="dept in deptList" 
      :key="dept.id" 
      :label="dept.name" 
      :value="dept.id" 
    />
  </el-select>
</template>
```

### 直接使用树形结构

如果需要保持树形结构（用于级联选择器等）：

```vue
<script setup>
import { getTrees as getDeptTrees } from '@/views/system/depts/api'

const deptTree = ref([])

getDeptTrees().then((res) => {
  deptTree.value = res.data || []
})
</script>

<template>
  <el-cascader 
    v-model="form.deptId" 
    :options="deptTree"
    :props="{ label: 'name', value: 'id', children: 'children' }"
    placeholder="请选择部门"
  />
</template>
```

## API接口说明

### 获取部门树
```
GET /api/system/dept/getTrees
```

**响应示例**:
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "name": "总部",
      "parentId": "0",
      "children": [
        {
          "id": 2,
          "name": "销售部",
          "parentId": "1",
          "children": [
            {
              "id": 8,
              "name": "大客户销售部",
              "parentId": "2",
              "children": []
            }
          ]
        }
      ]
    }
  ]
}
```

### 其他基础接口
- `GET /api/system/dept/list` - 获取部门列表
- `GET /api/system/dept/getOne/:id` - 获取单个部门
- `POST /api/system/dept/save` - 保存部门
- `POST /api/system/dept/add` - 新增部门
- `PUT /api/system/dept/update` - 更新部门
- `DELETE /api/system/dept/del/:ids` - 删除部门

## 注意事项

1. **部门树展平**: 由于后端返回的是树形结构，而el-select需要扁平列表，所以需要进行展平处理
2. **缓存优化**: 部门数据变化不频繁，可以考虑缓存部门列表以提高性能
3. **权限控制**: 确保用户有访问部门接口的权限
4. **数据完整性**: 删除部门时需要检查是否有子部门或关联的用户/客户
