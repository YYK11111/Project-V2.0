# 主题颜色数据库存储功能说明

## 功能概述

主题颜色设置现已支持存入数据库，实现跨设备、跨浏览器的主题同步。

## 实现方案

### 1. 数据库变更

在 `sys_user` 表中添加了 `theme_hsl` 字段：
```sql
ALTER TABLE `sys_user` 
ADD COLUMN `theme_hsl` VARCHAR(50) DEFAULT NULL COMMENT '主题颜色HSL值，格式：h,s,l' AFTER `nickname`;
```

### 2. 后端接口

#### 获取用户主题配置
- **接口**: `GET /api/system/users/getTheme`
- **返回**: `{ themeHsl: "345,82%,54%" }` 或 `{ themeHsl: null }`

#### 更新用户主题配置
- **接口**: `PUT /api/system/users/updateTheme`
- **参数**: `{ themeHsl: "345,82%,54%" }`
- **返回**: `{ success: true }`

### 3. 前端实现

#### 主题加载流程

1. **用户登录时**：
   - 调用 `getUserInfo()` 获取用户信息
   - 同时调用 `getTheme()` 获取主题配置
   - 应用主题到 CSS 变量
   - 同步到 localStorage（兼容旧数据）

2. **页面刷新时**：
   - Settings 组件从数据库重新获取主题
   - 如果数据库没有，则从 localStorage 恢复（兼容）

#### 主题保存流程

1. **用户选择颜色时**：
   - 实时更新 CSS 变量（即时预览）
   - 保存到 localStorage（临时存储）

2. **点击"确定"按钮时**：
   - 调用 `updateTheme()` 保存到数据库
   - 显示成功提示
   - 关闭设置面板

3. **点击"重置"按钮时**：
   - 恢复默认主题（H: 345, S: 82%, L: 54%）
   - 清空数据库中的主题配置
   - 清空 localStorage

### 4. 兼容性设计

- **向后兼容**：如果数据库中没有主题配置，会从 localStorage 读取
- **降级处理**：如果获取主题失败，使用 localStorage 或默认主题
- **双重存储**：同时维护数据库和 localStorage，确保可靠性

## 优势

✅ **跨设备同步**：在不同设备登录同一账号，主题自动同步  
✅ **持久化存储**：清除浏览器缓存后主题不会丢失  
✅ **团队统一**：可以为团队设置统一的主题规范  
✅ **用户体验**：登录即恢复个人偏好，无需重复设置  

## 测试建议

1. 登录系统，打开主题设置，选择一个颜色并点击"确定"
2. 刷新页面，验证主题是否保持
3. 退出登录，清除浏览器缓存
4. 重新登录，验证主题是否恢复
5. 在不同浏览器登录同一账号，验证主题是否同步
