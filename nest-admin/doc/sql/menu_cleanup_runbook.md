# 菜单清理执行说明

## 目标

清理以下高风险菜单配置，避免左侧导航空白、重复菜单和隐藏路由冲突继续出现：

1. 与前端常量隐藏路由重复的数据库隐藏菜单
2. `menu` 类型但缺少组件路径的菜单
3. 同级菜单下重复的路由地址
4. 没有可见子菜单且未配置组件的空目录

## 推荐执行顺序

1. 打开前端“菜单管理”页面，先查看体检摘要和体检明细
2. 优先处理高风险项：
   - 重复隐藏路由
   - 缺少组件
   - 同级重复路由
3. 再处理中风险项：
   - 空目录

## SQL 文件说明

### 1. `cleanup_redundant_hidden_route_menus.sql`

用途：清理与前端常量隐藏路由重复的数据库隐藏菜单。

执行步骤：

1. 先执行文件中的查询语句，确认候选菜单
2. 再确认这些菜单是否仍有 `sys_role_menu` 绑定
3. 备份 `sys_menu` 和 `sys_role_menu`
4. 确认无误后执行注释中的删除/软删除语句

### 2. `audit_invalid_menu_configs.sql`

用途：排查无组件菜单、空目录、重复 path。

执行步骤：

1. 先执行全部查询语句
2. 对照菜单管理页体检明细确认每条菜单用途
3. 对于重复 path：保留一个有效菜单，把角色绑定迁移到保留项，再删除其余菜单
4. 对于空目录：如果没有业务用途直接删除；如果应该承载页面，则补组件或补子菜单

### 3. `final_menu_cleanup_template.sql`

用途：把上面的排查结果落成最终清理步骤模板。

执行步骤：

1. 先按模板中的查询语句确认目标菜单
2. 将 `<menu_ids>`、`<keep_menu_id>`、`<drop_menu_ids>` 替换成真实值
3. 先迁移 `sys_role_menu`，再软删除 `sys_menu`
4. 执行模板末尾的验证 SQL，确认风险数量下降

## 风险项处理建议

### 重复隐藏路由

建议动作：删除数据库隐藏菜单，保留前端常量隐藏路由。

### 缺少组件

建议动作：

1. 如果它本来就是页面菜单：补齐组件路径
2. 如果它只是一个目录：改成 `catalog`
3. 如果它只是权限点：改成 `button`

### 同级重复路由

建议动作：

1. 保留一个菜单作为唯一入口
2. 将其余菜单的角色权限迁移到保留菜单
3. 删除重复菜单

### 空目录

建议动作：

1. 如果目录只是历史遗留：删除
2. 如果目录需要保留：补子菜单或补组件

## 验证步骤

完成任意一轮清理后，按下面顺序验证：

1. 前端打开菜单管理页，确认体检摘要下降
2. 使用目标账号登录，点击左侧导航确认不再出现空白
3. 执行前端检查：`npm run type-check`
4. 执行后端构建：`npm run build`

## 推荐批次

建议按下面批次处理，避免一次改动过大：

1. 第一批：重复隐藏路由
2. 第二批：缺少组件的 menu 菜单
3. 第三批：同级重复路由
4. 第四批：空目录
