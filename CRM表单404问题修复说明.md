# CRM表单404问题修复说明

## 问题描述
点击CRM模块的"新增客户"按钮后，页面跳转到404错误页。

## 根本原因

### 菜单结构差异
**项目管理模块（正常工作）**：
```
一级菜单（catalog）: 项目管理 (path: projectManage, component: business/projectManage/index)
  └─ 二级菜单（menu, hidden）: 项目表单 (path: projectManage/form, component: business/projectManage/form)
```

**CRM模块（之前错误）**：
```
一级菜单（catalog）: CRM客户管理 (path: /crm)
  └─ 二级菜单（catalog）: 客户管理 (path: customer, 无component)
      ├─ 三级菜单（menu）: 客户列表 (path: index, component: business/crm/customerManage/index)
      └─ 三级菜单（menu, hidden）: 客户表单 (path: form, component: business/crm/customerManage/form)
```

### 路由跳转逻辑
`RequestChartTable.goRoute()` 方法的路径拼接逻辑：
```javascript
const fullPath = path.startsWith('/') ? path : this.$route.path + '/' + path
```

当用户在 `/crm/customer/index` 页面点击"新增客户"时：
- `$route.path` = `/crm/customer/index`
- `path` = `'form'`
- 拼接结果：`/crm/customer/index/form` ❌ **错误路径**
- 正确路径应该是：`/crm/customer/form` ✅

## 解决方案

将CRM菜单结构改为与项目管理一致的2层结构：

**修复后的CRM结构**：
```
一级菜单（catalog）: CRM客户管理 (path: /crm)
  └─ 二级菜单（menu）: 客户管理 (path: customer, component: business/crm/customerManage/index)
      ├─ 三级菜单（menu, hidden）: 客户表单 (path: form, component: business/crm/customerManage/form)
      └─ 三级菜单（button, hidden）: 客户新增/编辑/删除
```

这样生成的路由为：
- `/crm/customer` → 列表页（自动重定向自 `/crm/customer/index`）
- `/crm/customer/form` → 表单页 ✅

## 技术要点

### Vue Router嵌套路由规则
- **catalog类型**：作为路由容器，使用 `<RouterView>` 渲染子路由
- **menu类型**：直接加载组件，不渲染子路由

### 路径拼接规则
当父级是catalog且有component时：
- 访问父级路径会自动加载其component
- 子路由通过相对路径拼接：`父路径 + '/' + 子路径`

### 最佳实践
所有带表单页的模块都应采用2层结构：
```
一级（catalog，指向列表页）
  └─ 二级（menu，隐藏，表单页）
```

避免使用3层结构（catalog → catalog → menu），这会导致路径拼接复杂化。
