# 菜单管理筛选问题修复说明

## 问题描述

在菜单管理页面筛选"菜单类型"为"菜单（menu）"时，列表显示为空白。

## 问题原因

### 1. 后端逻辑缺陷

**文件**: `nest-admin/src/modules/menus/menus.service.ts`

**原代码**:
```typescript
async list(query: QueryListDto = {}, isTree = true) {
  let { isActive, name, type } = query
  let queryOrm: FindManyOptions = {
    where: { isActive, type, name: this.sqlLike(name) },
  }

  let data = await this.repository.find(queryOrm)
  return isTree ? arrayToTree(data) : data
}
```

**问题分析**:
- 当筛选 `type='menu'` 时，只返回 menu 类型的数据
- `arrayToTree` 函数需要通过 `parentId` 构建树形结构
- 但父节点（catalog 类型）被 WHERE 条件过滤掉了
- 所有 menu 节点找不到父节点，被 `arrayToTree` 丢弃
- 最终返回空数组

**arrayToTree 函数逻辑**（来自 `src/common/utils/common.ts`）:
```typescript
export function arrayToTree(data, id = 'id', parentId = 'parentId', children = 'children', rootId = '0') {
  // ...
  data.forEach((item) => {
    let parent = map[item[parentId]]
    if (parent) {
      // 找到父节点，添加到父节点的 children 中
      parent[children] || (parent[children] = [])
      parent[children].push(item)
    } else if (!rootId || item[parentId] === rootId) {
      // 找不到父节点但 parentId === rootId，作为根节点
      result.push(item)
    }
    // 否则：找不到父节点且 parentId !== rootId，该节点被丢弃！
  })
  return result
}
```

### 2. CRM菜单父节点丢失

**数据库问题**:
- CRM子菜单（id=109, 114, 119, 124）的 `parent_id = 108`
- 但 id=108 的菜单已被删除（可能是之前的清理脚本误删）
- 这些菜单成为"孤儿节点"，无法正确显示

## 修复方案

### 修复1：后端增加父节点查询

**文件**: `nest-admin/src/modules/menus/menus.service.ts`

**修复代码**:
```typescript
async list(query: QueryListDto = {}, isTree = true) {
  let { isActive, name, type } = query
  let queryOrm: FindManyOptions = {
    where: { isActive, type, name: this.sqlLike(name) },
  }

  let data = await this.repository.find(queryOrm)
  
  // 如果按 type 筛选，需要同时查询父节点以正确构建树形结构
  if (type && data.length > 0) {
    const parentIds = [...new Set(data.map(item => item.parentId).filter(id => id && id !== '0'))]
    if (parentIds.length > 0) {
      const parents = await this.repository.find({
        where: { id: parentIds },
      })
      data = [...parents, ...data]
    }
  }
  
  return isTree ? arrayToTree(data) : data
}
```

**修复逻辑**:
1. 查询筛选条件的菜单数据
2. 如果有筛选条件且有数据，收集所有父节点ID
3. 查询这些父节点数据
4. 合并父节点和子节点数据
5. `arrayToTree` 就能正确构建树形结构

### 修复2：恢复CRM菜单父节点

**SQL修复**:
```sql
-- 更新所有CRM子菜单的父节点为CRM catalog(id=150)
UPDATE `sys_menu` 
SET `parent_id` = 150 
WHERE `parent_id` = 108;
```

**修复效果**:
- CRM子菜单现在正确指向父节点（id=150，CRM客户管理）
- 树形结构完整，可以正常显示

## 技术要点

### 树形结构构建原理
1. **数据准备**：需要同时有父节点和子节点数据
2. **映射创建**：使用 Map 存储所有节点（id -> node）
3. **关系建立**：遍历节点，通过 parentId 找到父节点
4. **添加子节点**：将子节点添加到父节点的 children 数组中
5. **收集根节点**：没有父节点或 parentId=rootId 的节点作为根节点

### 常见问题
- **孤儿节点**：parentId 指向不存在的节点，会被丢弃
- **循环引用**：A 的父节点是 B，B 的父节点是 A（会导致栈溢出）
- **断层的树**：中间节点缺失，导致子节点被提升为根节点

### 最佳实践
1. **删除检查**：删除菜单前检查是否有子菜单
2. **批量查询**：需要父节点时，批量查询而不是逐个查询
3. **软删除**：使用 is_delete 标记而不是物理删除，避免孤儿节点
4. **数据验证**：定期检查菜单数据的完整性
